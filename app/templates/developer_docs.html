OAuth 2.0 授权流程

1
用户发起授权请求
用户在第三方应用中点击"使用少年友晴天账号登录"按钮，第三方应用将用户重定向到授权页面。
GET /oauth/authorize?app_id=YOUR_APP_ID&redirect_uri=YOUR_CALLBACK_URL
2
用户登录并授权
用户在授权页面输入账号密码进行登录，确认授权第三方应用访问其基本信息。
3
系统生成授权码
用户授权成功后，系统生成一个授权码（auth_code），并通过回调地址将授权码返回给第三方应用。
YOUR_CALLBACK_URL?auth_code=AUTH_CODE_HERE
4
第三方应用获取用户信息
第三方应用使用授权码，通过后端 API 获取用户信息。
5
完成授权流程
第三方应用获取到用户信息后，可创建用户会话或绑定账号，授权流程完成。
重要提示
授权码有效期为10分钟，请在获取后尽快使用。每个授权码只能使用一次，使用后即失效。

回调机制说明

登录回调地址
当用户成功授权后，系统会将授权码通过此地址返回给第三方应用。您需要在应用配置中设置此地址。
https://your-app.com/callback?auth_code={auth_code}
验证码回调地址
当用户请求发送验证码时，系统会通过此地址将验证码信息回调给您。您可以在此接收并处理验证码，用于自定义的验证流程。
https://your-app.com/verification-callback?code={code}&user_id={user_id}&type={type}
回调参数说明
- code: 验证码内容
- user_id: 用户ID
- type: 验证码类型（email 或 phone）
- auth_code: 授权码（用于关联会话）
安全提示
请确保您的回调地址使用 HTTPS 协议，并做好接口的安全验证，防止被恶意调用。

API 接口文档

POST 获取用户信息
/api/oauth/userinfo
通过授权码获取已授权用户的基本信息。
参数名 类型 必填 说明
app_id string 是 应用的唯一标识符（15位）
app_secret string 是 应用密钥（64位）
auth_code string 是 授权码（10分钟内有效）
delete_code boolean 否 是否在获取用户信息后删除授权记录
cURL 
curl -X POST {{ project_website }}/api/oauth/userinfo \
  -H "Content-Type: application/json" \
  -d '{
    "app_id": "YOUR_APP_ID",
    "app_secret": "YOUR_APP_SECRET",
    "auth_code": "USER_AUTH_CODE"
  }'
Python 
import requests
import json

url = "{{ project_website }}/api/oauth/userinfo"
payload = {
    "app_id": "YOUR_APP_ID",
    "app_secret": "YOUR_APP_SECRET",
    "auth_code": "USER_AUTH_CODE"
}
headers = {"Content-Type": "application/json"}

response = requests.post(url, json=payload, headers=headers)
data = response.json()

if data.get("success"):
    user = data["user"]
    print(f"用户ID: {user['user_id']}")
    print(f"用户名: {user['username']}")
    print(f"头像: {user['avatar']}")
JavaScript 
const response = await fetch('{{ project_website }}/api/oauth/userinfo', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'

    app_id: 'YOUR_APP_ID',

  const user = data.user;
  console.log('用户ID:', user.user_id);
  console.log('用户名:', user.username);
  console.log('头像:', user.avatar);
}

POST 发送验证码
/api/oauth/send-verification
向已授权用户发送验证码，支持邮箱验证码和手机短信验证码两种方式。
参数名 类型 必填 说明
app_id string 是 应用的唯一标识符
app_secret string 是 应用密钥
auth_code string 是 有效的授权码
type string 是 验证码类型：email（邮箱）或 phone（手机）
cURL 
curl -X POST {{ project_website }}/api/oauth/send-verification \
  -H "Content-Type: application/json" \
  -d '{
    "app_id": "YOUR_APP_ID",
    "app_secret": "YOUR_APP_SECRET",
    "auth_code": "USER_AUTH_CODE",
    "type": "email"
  }'
Python 
import requests
import json

url = "{{ project_website }}/api/oauth/send-verification"
payload = {
    "app_id": "YOUR_APP_ID",
    "app_secret": "YOUR_APP_SECRET",
    "auth_code": "USER_AUTH_CODE",
    "type": "email"
}
headers = {"Content-Type": "application/json"}

response = requests.post(url, json=payload, headers=headers)
data = response.json()

if data.get("success"):
    print("验证码发送成功")
else:
    print(f"发送失败: {data.get('message')}")

频率限制
每个应用对同一用户发送验证码的频率限制为每分钟1次，请合理控制发送频率。

错误码说明

错误码 HTTP状态码 说明 解决方案
缺少必要参数 400 请求参数不完整 检查并补充所有必填参数
app_secret验证失败 401 应用密钥错误 检查 app_secret 是否正确
授权码无效 404 授权码不存在或已过期 重新获取授权码
授权码已过期 401 授权码超过10分钟有效期 重新进行授权流程
应用不存在或未通过审核 404 应用ID不存在或未审核通过 检查应用状态或提交审核
发送频率限制 429 验证码发送过于频繁 等待60秒后重试
